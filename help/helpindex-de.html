(clef) Text(Noten)


name

Der Name des Stücks, in den meisten Fällen das incipit, die ersten Worte. Im Falle eines Messordinariums wird z. B. Kyrie X Alme Pater oder Sanctus XI als Titel empfohlen.

gabc-copyright

Diese Lizenz ist die Copyright-Notiz (englisch zu verfassen) für die gabc-Datei, je nachdem, unter welcher Lizenz der Schreiber (dessen Name in das Feld gabc-Copyright einzutragen ist)die Datei veröffentlichen möchte. Außer der Lizenz kann dieses Feld eine kurze Beschreibung der Lizenz enthalten, so wie etwa public domain, CC-by-sa; die vollständige Lizenz sollte in einer eigenen Textdatei untergebracht werden. Zu den gesetzlichen Bestimmungen, die für gregorianische Notation zutreffen, lesen Sie bitte die dafür eingerichtete Seite auf diesem Server. Ein Beispiel, wie dieses Feld aussehen kann:

(C) Elie Roux, 2009, public domain

score-copyright

Dieses Feld enthält die Copyright-Notiz (englisch) der verwendeten Quellnoten, aus denen die gabc-Datei entstanden ist. Wie bei der gabc-Lizenz gibt es auch hier die Möglichkeit einer kurzen Beschreibung. In unklaren oder komplexen Fällen kann dieses Feld leer gelassen werden. Am Besten ist es, wenn der Copyright-Inhaber und der Lizenzierer des Stücks eine Person sind. Auch hier siehe die Seite zur Gesetzeslage. Ein Beispiel, wie dieses Feld aussehen kann:

(C) Abbaye de Solesmes, 1934

office-part

Dieses Feld beinhaltet die liturgische Kategorie für das Stück, beispielsweise: antiphona, hymnus, responsorium brevium, responsorium prolixum, introitus, graduale, tractus, offertorium, communio, kyrie, gloria, credo, sanctus, benedictus, agnus dei.

occasion

Dieses Feld beinhaltet Informationen über die Verortung im liturgischen Kalender, beispielsweise Dominica II Adventus, Commune doctorum, Feria secunda.

meter

FÃ¼r Hymnen oder andere Stücke mit sich wiederholenden Strophen wird hier das Metrum eingetragen, die Zahl der Silben in jeder Zeile einer Strophe. Für typische ambrosianische Hymnen wäre dies beispielsweise 8.8.8.8: Vier Zeilen von je acht Silben.

commentary

Hier können Kommentare eingefügt werden, beispielsweise die verwendete Bibelstelle für den Text.

arranger

Der Name eines modernen Arrangierers, wenn ein traditioneller Choral auf einen neuen Text adaptiert wurde oder ein Manuskript in Quadratnotation überführt wurde. Dies kann auch ein Firmenname sein. Beispiel: Solesmes.

gabc-version

Die Version von gabc, die zum Setzen benutzt wurde. Zur Zeit sollte dies dieselbe Version wie die gregorio-Version sein.

author

Der Autor des Stücks, soweit bekannt; natürlich sind die meisten Autoren für Choralstücke unbekannt.

date

Das Datum der Komposition oder seiner ältesten überlieferten Handschrift. Bei den meisten Stücken wird das lediglich annähernd bestimmbar sein wie z. B. XI. s. für 11. Jahrhundert. Die Konvention gibt vor, lateinische Zahlen zu benutzen, so wie im vorangehenden Beispiel.

manuscript

FÃ¼r Transkriptionen direkt aus einem Manuskript der Name der Handschrift, beispielsweise Montpellier H.159.

manuscript-reference

Eine eindeutige Referenz für das Zück, nach gut bekannten Systemen. Beispielsweise kann die Referenz mit cao (in der Cantus database of office chants) beginnen. Wenn die Referenz unklar in Bezug auf das benutzte System ist, sollte der Name des Systems vorangestellt werden. Die Referenz sollte das Stück, nicht das gesamte Manuskript, bezeichnen; alles, was das gesamte Manuskript bezeichnet, gehört in das Feld manuscript.

manuscript-storage-place

Für Trankriptionen direkt aus einem Manuskript: der Ort, an dem das Manuskript sich befindet; beispielsweise Bibliothéque Nationale, Paris.

book

Für Transkriptionen aus einer modernen Ausgabe (wie diejenigen von Solesmes); modern bezieht sich auf Ausgaben mindestens bis zum 19. Jahrhundert. Das Feld sollte den Namen dieser Ausgabe beinhalten, wie Liber Usualis.

transcriber

Der Name des Transkriptors.

transcription-date

Das Datum der Transkription, in der folgenden Form: jjjjmmtt, wie 20090129 für den 29. Januar 2009.

gregoriotex-font

Die Schrift, die von GregorioTeX benutzt wird; zur Zeit sind drei Schriften kompatibel: greciliae, gregorio und parmesan.

mode

Der Ton des Stücks, normalerweise eine arabische Ziffer zwischen 1 und 8 bzw. jeder beliebige Text für andere Fälle. Dieser Text kommt nicht über die Initiale, dafür siehe annotation.

initial-style

Der Stil der Initiale. 0 bedeutet keine Initiale, 1 eine "normale", 2 eine große über zwei Zeilen. Beachten Sie, dass Sie die ersten zwei Zeilenumbrüche manuell festlegen sollten, wenn die Initiale über zwei Zeilen gehen soll.

user-notes

Hier kann jeder beliebige Text hinein, der die anderen Header-Felder ergänzt. Soweit möglich sollten aber für die Informationen zum Stück die vorgesehenen Felder ausgefüllt werden, um ein bestimmtes Stück leichter auffinden zu können.

annotation

Hier wird der Text über der Initiale eingefügt, üblicherweise eine Abkürzung der liturgischen Verortung in der ersten Zeile und eine Bezeichnung des Modus in der zweiten Zeile. Bis zu zwei dieser Felder können benutzt werden; werden zwei benutzt, ist das erste die obere, das zweite die untere Zeile über der Initiale. Ein Beispiel:

annotation:Ad Magnif.;

annotation:VIII G;

generated-by

Name der Software, die die gabc-Datei erstellt hat. Wenn Sie sie selbst verfasst haben, sollte dieses Feld nicht verwendet werden.

style

Der typographische Stil der Musik. Bisher noch nicht unterstützt, daher ist dieser Eintrag (noch) funktionslos. Beispiele: editio vaticana, editio medicea.

virgula-position

Position der Virgula. Noch nicht unterstüzt, daher ist dieser Eintrag (noch) funktionslos. Beispiele: top, bottom.

number-of-voices

Anzahl der Stimmen in einem polyphonen Stück

Nach dem Separator %% folgen die Noten: (clef) text(notes). Im Folgenden soll es nur um diesen Teil der Datei gehen, das Kernstück der Notation.
